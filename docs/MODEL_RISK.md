# 回测口径与模型风险

本文件说明当前工程能证明什么、不能证明什么。它是实盘上线前的强制检查清单。

## 当前结果的适用范围

- `configs/universe.json` 与 `data/history_api` 当前配套的是 deterministic synthetic 数据，代码中的 `RB2405`、`IF2406` 等仅是演示标签。它们在 2025—2026 年已经不是可交易合约，因此所有现有收益、Sharpe、排名只验证程序链路。
- `optimize-strategy` 严格隔离最终测试段，可用于检验选参流程是否泄漏；synthetic 数据本身含人工规律，样本外表现仍不能转换为真实收益预期。
- 成交信号在当前 K 线收盘后形成，并在下一根 K 线开盘成交；期末持仓计入退出滑点和手续费。

## 真实数据接入后仍需处理

1. **逐合约与换月**：信号连续价格和实际成交合约必须分离，记录主力切换、换月日与换月成本，并校验上市/到期日期。
2. **共享账户**：当前 `batch-backtest` 是多个独立子账户净值的等权研究结果，不是共享 500 万元账户；它尚未统一计算跨品种保证金、板块敞口和账户级强平。
3. **币种折算**：CL、GC、ES 等美元盈亏尚未按历史 USD/CNY 汇率折算。混合市场组合前必须增加逐日 FX 数据。
4. **标的类型**：`binance` provider 当前读取现货 K 线。现货不能直接套用期货做空、保证金和资金费率口径；研究比特币期货/永续时应通过 `http` provider 接入对应合约数据和资金费率。
5. **交易所规则**：尚未完整模拟涨跌停不可成交、历史保证金/手续费变化、平今费、限仓、交割月禁入、异常撤单和强平撮合。
6. **交易日历**：国内夜盘应归属下一交易日；BTC、国内期货和海外期货不能共用一个固定年化因子或交易日历。
7. **容量与冲击**：固定 ticks 滑点只是基础压力测试，真实上线前还需按成交量、盘口深度和参与率建立冲击成本模型。

## 真实回测最低门槛

- 使用足够长的真实逐合约行情，并保存原始数据、清洗版本、数据源和下载时间。
- 在最终测试段封存前确定候选策略、参数范围、目标函数和成本模型。
- 同时报告训练、验证、最终测试、成本翻倍、参数邻域和分品种结果；不以单一最高收益作为上线依据。
- 先接只读行情和仿真账户，核对交易软件账单、保证金、手续费、持仓和本项目成交日志。
- 在共享账户、换月、币种、涨跌停和强平规则完成前，不把当前结果用于实盘仓位决策。
